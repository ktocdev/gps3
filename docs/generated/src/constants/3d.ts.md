# 3D Environment Shared Constants

`src/constants/3d.ts`

> Centralizes all configuration values used across the project's 3D composables and rendering code. It exists to ensure consistency and ease of maintenance by providing a single source of truth for grid layout, camera behavior, lighting, colors, animation, clouds, and item models.

This file exports a set of frozen (`as const`) constant objects, each grouping related 3D configuration values.

### Key exports
- **GRID_CONFIG**: grid dimensions (`COLS`, `ROWS`, `CELL_SIZE`, `PIXELS_PER_CELL`) with computed getters `WIDTH` (COLS × CELL_SIZE) and `DEPTH` (ROWS × CELL_SIZE).
- **CAMERA_CONFIG**: field of view, near/far planes, initial position, height/position bounds, and various movement/zoom/rotation speed factors plus tilt offset tuning.
- **SCENE_COLORS**: hex color values for sky, ambient, directional, and back lighting.
- **LIGHTING_CONFIG**: light intensities, shadow map size, and shadow camera bounds.
- **ENVIRONMENT_CONFIG**: wall dimensions and bedding texture/piece counts.
- **GUINEA_PIG_CONFIG**: default fur/marking/ear/eye colors and material roughness/clearcoat settings.
- **ANIMATION_CONFIG**: movement threshold and selection ring geometry/pulse parameters.
- **CLOUD_CONFIG**: two cloud layers (HIGH_LAYER, LOW_LAYER) with counts, heights, distances, scale, plus shared drift/color/opacity and puff/cluster settings.
- **ITEM_CONFIG**: hay color palette and instance counts/dimensions, wood texture size, and food positions within a bowl.

All objects are declared with `as const` for literal type narrowing and immutability. GRID_CONFIG uses TypeScript getters for derived values.
